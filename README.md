# Airport Baggage Counter

## Описание проекта

**Airport Baggage Counter** — веб-приложение для автоматического обнаружения и подсчёта багажа на изображениях транспортной ленты аэропорта с использованием технологий компьютерного зрения и искусственной нейронной сети **YOLOv8**.

Проект разработан в рамках практической работы по дисциплине, посвящённой применению искусственных нейронных сетей для решения задач компьютерного зрения.

---

## Основные возможности

- загрузка изображений через веб-интерфейс;
- автоматическая обработка изображений с использованием модели YOLOv8;
- обнаружение объектов багажа (чемоданы, рюкзаки и сумки);
- отображение bounding boxes вокруг найденных объектов;
- подсчёт количества обнаруженного багажа;
- отображение времени обработки изображения;
- сохранение истории обработки в формате JSON;
- экспорт истории обработки в Microsoft Excel.

## Установка проекта

### 1. Клонирование репозитория

```bash
git clone https://github.com/<USERNAME>/airport_baggage_counter.git
```

### 2. Переход в каталог проекта

```bash
cd airport_baggage_counter
```

### 3. Создание виртуального окружения

```bash
python3 -m venv venv
```

### 4. Активация виртуального окружения

**macOS / Linux**

```bash
source venv/bin/activate
```

**Windows**

```cmd
venv\Scripts\activate
```

### 5. Установка зависимостей

```bash
pip install -r requirements.txt
```

## Использование

1. Открыть приложение в браузере.
2. Выбрать изображение с багажом.
3. Нажать кнопку **«Запустить обработку»**.
4. Дождаться завершения анализа.
5. Просмотреть:
   - количество обнаруженного багажа;
   - время обработки;
   - обработанное изображение с bounding boxes.
6. При необходимости скачать историю обработки в формате Excel.

---

## Принцип работы

1. Пользователь загружает изображение.
2. Flask принимает HTTP-запрос.
3. OpenCV считывает изображение.
4. YOLOv8 выполняет детектирование объектов.
5. Выполняется фильтрация объектов классов:
   - suitcase;
   - backpack;
   - handbag.
6. Подсчитывается количество найденных объектов.
7. Формируется изображение с bounding boxes.
8. Результат сохраняется.
9. История обработки записывается в файл `history.json`.
10. При необходимости формируется отчёт `history.xlsx`.

## Используемая модель

В проекте применяется предварительно обученная модель:

**YOLOv8n**

Модель загружается автоматически библиотекой **Ultralytics** и используется без дополнительного обучения.

---

## Ограничения

Разработанная система имеет следующие ограничения:

- поддерживается обработка отдельных изображений;
- распознаются только объекты, присутствующие в модели COCO;
- в качестве багажа учитываются классы:
  - suitcase;
  - backpack;
  - handbag;
- качество обнаружения зависит от разрешения изображения, освещения и степени перекрытия объектов.
